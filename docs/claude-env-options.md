# Claude Code Environment Options & Best Practices

CodeConductor configures Claude Code with specific environment variables and settings to optimize performance, security, cost, and developer experience. These configurations represent engineering best practices when integrating Claude Code into a team's workflow.

Here is a breakdown of the active environment variables configured in `settings.json` for supported presets (e.g., `claude`, `seo-hotel`):

## Environment Variables (`env`)

| Variable | Recommended Value | Description | Why it is a Best Practice |
|----------|-------------------|-------------|---------------------------|
| `CLAUDE_CODE_ATTRIBUTION_HEADER` | `"0"` | Disables co-author attribution and headers in git commits generated by Claude. | Keeps commit logs clean, readable, and standard (following conventional commit styles). |
| `CLAUDE_CODE_DISABLE_1M_CONTEXT` | `"1"` | Limits Claude's maximum context size in active memory. | Prevents runaway context inflation, saving token costs and reducing response latency. |
| `CLAUDE_CODE_DISABLE_AUTO_MEMORY` | `"1"` | Disables Claude's automatic local memory storage across turns/sessions. | Ensures deterministic behavior across runs, preventing past sessions or unrelated memories from polluting the current task context. |
| `CLAUDE_CODE_DISABLE_NONESSENTIAL_TRAFFIC` | `"1"` | Disables telemetry and non-essential network traffic sent by the agent. | Increases privacy, improves performance, and minimizes network overhead. |
| `CLAUDE_CODE_DISABLE_TERMINAL_TITLE` | `"1"` | Prevents Claude from rewriting the terminal / console title window. | Avoids visual disruption in terminal emulators, Zellij/Tmux panes, and IDE terminals. |
| `DISABLE_EXTRA_USAGE_COMMAND` | `"1"` | Disables verbose command suggestions or help hints at the end of turns. | Reduces output verbosity, keeping the developer's log clean and concise. |
| `CLAUDE_CODE_SIMPLE_SYSTEM_PROMPT` | `"1"` | Uses a simpler, stripped-down system prompt for Claude. | Saves prompt tokens and keeps the agent focused on executing direct commands without prompt bloat. |
| `ENABLE_TOOL_SEARCH` | `"auto"` | Controls automatic search behavior for tools and commands. | Balancing tool search capability with efficiency. |
| `CLAUDE_AUTOCOMPACT_PCT_OVERRIDE` | `"80"` | Directs Claude to trigger context compaction when memory reaches 80% capacity. | Manages context budget actively, maintaining prompt effectiveness without manual `/compact` calls. |
| `DISABLE_BUG_COMMAND` | `"1"` | Disables the automatic buggy/error command reporting interface. | Avoids prompt interruptions or unsolicited diagnostic screens during ordinary errors. |

## Settings & Permissions Best Practices

In addition to environment variables, our Claude integration enforces:

1. **Explicit Permission Constraints**:
   - **`allow`**: Safe operations such as `git status`, `git diff`, `npm test`, read operations, and directory-specific writes.
   - **`deny`**: Dangerous actions are blocked outright (e.g., `rm -rf`, force-pushing to protected branches, reading `.env` or sensitive secret directories).
   - **`ask`**: Actions requiring explicit user validation (e.g., `git push`, `npm install`, docker compose commands).

2. **Automated Tool Hooks**:
   - **Post-use formatting/linting**: Runs `prettier`, `eslint`, or `ruff` automatically on changed files (TS, JS, Python) to maintain codebase style consistency.
   - **Pre-use security checks**: Blocks terminal commands attempting to read `.env`, SSH keys, or credential files.

3. **Disabled survey prompts**: `feedbackSurveyRate` is set to `0` to keep the CLI non-interactive and scriptable.
